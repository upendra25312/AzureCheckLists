# Azure Architecture Review Board (ARB) Design Document Checklist

Use this checklist to review HLD/LLD documents for Azure solutions. Each area aligns with the frameworks and value proposition described in your platform purpose.

---

## 1. Well-Architected Framework (WAF)
- [ ] Reliability
- [ ] Security
- [ ] Cost Optimization
- [ ] Operational Excellence
- [ ] Performance Efficiency

## 2. Cloud Adoption Framework (CAF)
- [ ] Strategy & Planning
- [ ] Governance
- [ ] Management
- [ ] Migration & Innovation

## 3. Azure Landing Zone (ALZ)
- [ ] Subscription & Management Group Structure
- [ ] Policy & Blueprint Alignment
- [ ] Resource Organization

## 4. High Availability / Disaster Recovery (HA/DR)
- [ ] Redundancy & Failover
- [ ] Backup & Restore
- [ ] Geo-Replication

## 5. Backup
- [ ] Backup Policy
- [ ] Retention & Recovery Testing

## 6. Security
- [ ] Identity & Access Management (AAD/Entra ID)
- [ ] Data Protection & Encryption
- [ ] Secrets Management
- [ ] Regulatory Compliance

## 7. Networking
- [ ] Topology & Segmentation
- [ ] Firewalling & DDoS Protection

## 8. Monitoring
- [ ] Logging & Alerting
- [ ] Health Checks
- [ ] Cost Monitoring

## 9. Governance
- [ ] Policy Enforcement
- [ ] RBAC
- [ ] Resource Tagging & Organization

## 10. Cost Management & Optimization
- [ ] Budgeting & Cost Alerts
- [ ] Reserved Instances & Rightsizing

## 11. DevOps & Automation
- [ ] CI/CD Pipelines
- [ ] Infrastructure as Code
- [ ] Deployment Automation

## 12. Application Architecture & Design
- [ ] Scalability & Modularity
- [ ] Technology Choices
- [ ] Integration Patterns

## 13. AI/ML & Data Workloads (if applicable)
- [ ] Model Lifecycle Management
- [ ] Data Residency & Security
- [ ] Responsible AI Practices

---

**Instructions:**
- For each area, check off items that are addressed in the design document.
- Link findings to specific document sections and Microsoft Learn references where possible.
- Use this checklist for both quick scoping and full ARB-grade reviews.
