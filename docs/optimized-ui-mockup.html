Homepage direction

From first glance to sign-off, the product feels sharper, calmer, and more obvious.

Mode strategy

The optimized website separates the jobs cleanly.

The homepage should not make new users learn internal product structure. It should route them into the right path, make the value obvious, and keep advanced workflow depth tucked behind the point where it matters.

Job-first IA

Primary path

Standard review workspace

This becomes the main entry for most users. It keeps service scope, findings, pricing, evidence notes, and exports inside a single guided flow that feels fast rather than procedural.

Best for most architecture reviews Fast first pass Outputs visible early
  • Start review from the hero without filling a form immediately.
  • Keep a compact summary rail visible during the whole review.
  • Make service selection, findings, and pricing feel like one coherent workspace.

Advanced path

ARB-grade review mode

ARB should feel like a stricter lane inside the same product, not a separate confusing product. It gets stronger evidence handling, upload-first workflow, and decision-ready closeout.

Evidence-driven Reviewer-owned Decision-aware
  • Position ARB as opt-in rigor, not the first thing every user sees.
  • Keep AI recommendation separate from human decision and sign-off.
  • Show upload, evidence mapping, scorecard, and conditions as one lane.
